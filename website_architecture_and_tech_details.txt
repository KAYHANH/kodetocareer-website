4. BACKEND ARCHITECTURE — API ENDPOINTS & LOGIC
--------------------------------------------------------------------------------
The backend is powered by Next.js Server Actions and Route Handlers:

* LEAD CAPTURE ENDPOINT (`/api/enroll`):
  - File: `src/app/api/enroll/route.ts`
  - Logic: 
    1. Receives data (name, phone, qualification, status, year, courseTitle).
    2. Validates fields (checks if any required value is missing).
    3. Backup: Reads and appends data to a local `enrollments.json` file.
    4. Integration: Checks for `GOOGLE_SHEET_WEBAPP_URL` inside server variables.
    5. Apps Script Integration: Forwards enrollment payloads directly via HTTP 
       POST to Google Sheets for live sheet storage.
    6. Returns JSON: `{ success: true }` or status 400/500 errors.

5. REDIRECTS, STATES & NAVIGATION WORKFLOWS
--------------------------------------------------------------------------------
* Sticky Header: Transition state checks window scroll Y-axis. Opens overlay
